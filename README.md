# Occupational-Specialism

**1. Visual/Interface Design**

1. Create high-fidelity wireframes: 
   Include layout, navigation and user interactions. 
   ANNOTATE, explain design choices and accessibility features.
   Apply principles of: Layout and white space, Visual hierarchy, Common Conventions (navigation, information, symbols etc.).
   INCLUDE accessibility features.
   Must show understanding of: data validation, input masks, type casting, feedback to user/error messages
   
**2. Data Requirements**

1. Entity Relationship Diagrams:
    Show entities, attributes and relationships
    Label Primary and Foreign keys properly.
    Structure appropriately
    
2. Use Case Diagrams:
    Display system functionality
    Ensure functional requirements align with user needs
    Communicate design properly
    Include labels for each actor
    Ensure layout is clean

3. Class Diagrams:
    Show object-oriented structure, attributes and methods
    DECOMPOSE
    Ensure attributes, classes and methods are appropriate.

4. Data Flow Diagrmams:
    Ensure data flow is accurate
    Include relevant labels and processes
    Add appropriate External Entities

5. Data Dictionary:
    List data elements, types, constraints and validation
    Provide Examples
    Ensure data is labelled correctly
    Formatting should be valid
